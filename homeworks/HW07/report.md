## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- **Где KMeans "ломается" и почему?**
  - На датасете B с нелинейной структурой: silhouette всего 0.387 vs 0.521 у DBSCAN
  - Причина: KMeans предполагает шарообразные кластеры, а данные имеют сложную форму

- **Где DBSCAN/иерархическая кластеризация выигрывают и почему?**
  - DBSCAN выиграл на датасете B благодаря:
    - Способности находить кластеры произвольной формы
    - Автоматическому определению числа кластеров
    - Устойчивости к выбросам (помечает их как шум)

- **Что сильнее всего влияло на результат?**
  1. Масштабирование - критично для distance-based методов
  2. Структура данных (линейная/нелинейная)
  3. Наличие выбросов (влияет на DBSCAN через noise_ratio)
  4. Плотность кластеров (проблема для KMeans на DS3)

### 5.2 Устойчивость (обязательно для одного датасета)

- Какую проверку устойчивости делали: 5 запусков KMeans на DS1 с разными random_state (42-46)

  - Все запуски дали практически идентичные разбиения
- Вывод: KMeans на DS1 очень устойчив. Высокий ARI (>0.97) показывает, что решение практически не зависит от инициализации.

## 6. Conclusion

1. **Препроцессинг обязателен**: Масштабирование критически важно для distance-based методов

2. **Нет универсального метода**: 
   - KMeans хорош для шарообразных кластеров
   - DBSCAN лучше для сложных форм и работы с выбросами

3. **Метрики помогают, но не заменяют визуализацию**: 
   - Silhouette полезен для сравнения, но PCA дает понимание структуры

4. **Параметры важны**: 
   - Для KMeans: выбор k через elbow/silhouette метод
   - Для DBSCAN: аккуратный подбор eps и min_samples

5. **Устойчивость проверяема**: 
   - Многократные запуски с разной инициализацией показывают надежность решения

6. **Интерпретация возможна**: 
   - Даже без истинных меток можно анализировать профили кластеров

7. **Протокол эксперимента важен**: 
   - Четкая последовательность: EDA → препроцессинг → подбор параметров → оценка → визуализация

8. **Unsupervised требует критического мышления**:
